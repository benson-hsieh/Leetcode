## Description
You are given an integer array `nums`. The score of an index `i` is defined as the number of indices `j` such that `i < j < n` and `nums[i]` and `nums[j]` have different parity (one is even and the other is odd). Return an integer array `answer` where `answer[i]` is the score of index `i`.

## Approach: Brute Force with Nested Loops
For each element at index `i`, we iterate through all subsequent elements at index `j` (where $j > i$) and check if their parity is different from `nums[i]`.

### Key Steps:
1.  Initialize a result array `res` of the same length as `nums`.
2.  Use a nested loop:
    - Outer loop `i` from `0` to `n-1`.
    - Inner loop `j` from `i+1` to `n-1`.
3.  In the inner loop, use the modulo operator (`% 2`) to check if `nums[i]` and `nums[j]` have different parity.
4.  Increment `res[i]` whenever a pair with opposite parity is found.

## Complexity Analysis
- **Time Complexity**: O(N^2), where N is the length of the array, due to the nested loops.
- **Space Complexity**: O(1) extra space, excluding the space required for the output array.

## Solution (Java)

```java
class Solution {
    public int[] countOppositeParity(int[] nums) {
        int n = nums.length;
        int[] res = new int[n];
        for(int i = 0; i < n; i++){
            int num = nums[i] % 2;
            for(int j = i+1; j < n; j++){
                if(nums[j] % 2 != num)
                    res[i]++; 
            }
        }
        return res;
    }
}
